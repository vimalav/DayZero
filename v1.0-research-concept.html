Executive Summary

Three user research interviews revealed critical insights about the watsonx Orchestrate onboarding experience. The research identified integration challenges and time to value as the most significant barriers, with users requiring 10-30 minutes to understand product value (target: under 5-10 minutes).

🔴 Critical Finding: Integration Challenges

The #1 Blocker

Connections to external systems is the most significant barrier users face. AI engineers often lack credentials to create connections (ServiceNow, Salesforce, SAP, Workday, Milvus), leaving them completely blocked from building agents.

"The AI engineer typically does not have the credentials to connect to the back end systems... they're stuck. They can't do anything right." — Joseph Kozhaya, Client Engineering

Key Issues

  • Unclear ownership: Who creates connections (admin vs AI engineer)?
  • No RBAC separation causes role confusion
  • AI engineers lack visibility into existing connections
  • No clear workflow for connection setup

Immediate Actions

  1. Clarify connection workflow and admin vs builder roles
  2. Provide visibility into existing connections
  3. Create integration cookbooks for common systems
  4. Consider connection templates or guided setup

💡 Top Actionable Recommendation

Replace "Chat with Docs" with Client Zero Patterns

Users consistently said Client Zero patterns (Ask HR, Ask IT, Ask Procurement) help them understand agentic capabilities, while "chat with docs" feels outdated and doesn't showcase true agent functionality.

"A lot of clients resonate with the client 0 strategy... they see a demo and ask procurement, ask HR and they kind of get it." — Cameron Seitz, Client Engineering
"Chat with documents... people think it's old news and they don't think necessarily it's agents." — Joseph Kozhaya, Client Engineering

🎯 Landing Page Redesign

Current Issues

  • 4 tiles create decision paralysis
  • Confusion between "Create my first agent" vs "Guide me to create an agent"
  • "Ask Documentation" takes valuable space
  • No time estimates or complexity indicators

Recommended Structure: 3 Tiles

🤖 Create Agent

Build from templates or start fresh

  • Ask HR
  • Ask IT
  • Ask Procurement
  • Custom build

⏱️ 10-30 min | 📊 Beginner+

📚 Agent Catalog

Browse 300+ pre-built agents

  • Domain agents
  • Integration-specific
  • Filter by use case

⏱️ 2-5 min | 📊 All levels

🔗 Bring My Own Agent

Import & govern external agents

  • Import agents
  • Add governance
  • Evaluate performance

⏱️ 15-20 min | 📊 Intermediate

Why "Bring My Own Agent" Matters

"A lot of people are kind of coming in with external agents. They're not building with Orchestrate... that's almost like the new positioning because there's so many different apps that create agents nowadays." — Cameron Seitz

This addresses the growing market of external agents and highlights Orchestrate's governance capabilities as a key differentiator.

🎨 User Experience Improvements

Add Metadata to All Options

Users need to know what they're committing to before they start:

  • ⏱️ Estimated time (e.g., "5-10 minutes", "20-30 minutes")
  • 📊 Complexity level (Beginner/Intermediate/Advanced)
  • 🎯 Use case category (e.g., "Client Zero", "Document Processing")
  • 🔌 Integrations required (if any)

Skill-Level Personalization

"Maybe we could even do like when you first get on the page you give like a rating one to five how familiar you are with agentic tools or AI and then you know if someone gives A1... maybe don't show that option or show a more advanced option." — Cameron Seitz

Recommended approach: Quick 3-question survey on first login:

  1. How familiar are you with AI agents? (New / Some experience / Very experienced)
  2. What's your primary role? (Admin / AI Engineer / Business User)
  3. Coding preference? (No-code / Low-code / Pro-code)

👥 Role-Based Considerations

Critical Gap: No RBAC Separation

Currently, everyone sees everything, causing confusion. Different personas have different needs:

Persona Primary Tasks Key Needs
Admin Connections, models, analytics, credentials, governance System health, usage metrics, credential management
AI Engineer Building agents, testing, integrations, configuration Quick agent creation, integration tools, testing environment
Beginner Learning, exploring, first agent Tutorials, templates, guided experiences, examples

Analytics Section Placement

Current issue: Analytics is prominent on day-zero experience

Feedback: "Analytics... relevant for admins, not AI engineers... not at this point. I don't think they're interested yet, especially if they haven't built anything yet." — Joseph Kozhaya

Recommendation: Collapse by default for new users, expand for returning users
