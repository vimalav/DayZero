## What Went Wrong

### Technical Issues

1. **JavaScript Error**: `Cannot set properties of null (setting 'innerHTML')`
   - The `templatesGrid` element ID doesn't exist in the HTML
   - Should be `templateGrid` (singular) not `templatesGrid` (plural)
   - This breaks the entire rendering pipeline

2. **Modal Not Appearing**: Domain selection modal should show on load but doesn't
   - Modal HTML structure exists but JavaScript fails before reaching it
   - Error in template rendering prevents modal from displaying

3. **Missing HTML Elements**: Several elements referenced in JavaScript don't exist in HTML
   - `templatesGrid` → should be `templateGrid`
   - Templates section not properly structured

## Answering Your Specific Questions

### Q1: How does the landing page change based on user role?

**Answer:** In v4.2, the landing page changes in TWO ways:

1. **By Persona (Role):**
   - **Builder**: "Get started with your first agent" + 5-step agent creation checklist
   - **Admin**: "Set up your watsonx Orchestrate tenant" + 5-step admin setup checklist
   - **SME**: "Create your first workflow" + 5-step workflow creation checklist

2. **By Domain (NEW in v4.2):**
   - **Customer Care**: Ticket routing checklist + Customer Care templates
   - **Procurement**: PO approval checklist + Procurement templates
   - **HR**: Employee onboarding checklist + HR templates
   - **IT Support**: Incident management checklist + IT Support templates

**Example Combinations:**

- Builder + Customer Care = Agent creation checklist with ticket routing focus
- Admin + Procurement = Tenant setup checklist with ERP integration focus
- SME + HR = Workflow creation checklist with employee onboarding focus

### Q2: What is the purpose of asking questions about connections?

**Answer:** This is confusing because v4.2 has TWO different flows that got mixed up:

**OLD Flow (v4.0):**

- Questionnaire modal asks about connections
- Used to populate "Needs Attention" section
- Example: "Do you have API keys?" → If No → "API key missing" in Needs Attention

**NEW Flow (v4.2 - SHOULD BE):**

- Domain selection modal (NOT questionnaire)
- No questions about connections
- Smart recommendations detect connections automatically
- Example: System detects Zendesk → Shows "Zendesk detected" recommendation

**The Problem:** v4.2 still has the old questionnaire code, which conflicts with the new domain selection approach.

### Q3: Why have both "Needs Attention" and "Get Started" checklist?

**Answer:** They serve different purposes:

**"Needs Attention" (Red Banner):**

- **Purpose**: Urgent blockers that prevent progress
- **When it appears**: Only when something is broken or missing
- **Example**: "API key missing" - blocks step 2 of checklist
- **User action**: Must fix before continuing
- **Mental model**: "Something is wrong, fix it now"

**"Get Started" Checklist (Left Panel):**

- **Purpose**: Sequential steps to complete onboarding
- **When it appears**: Always visible
- **Example**: "1. Choose your agent type" → "2. Connect tools" → etc.
- **User action**: Follow steps in order
- **Mental model**: "Here's what to do next"

**Why Both?**

```
Normal Flow:
User → Step 1 → Step 2 → Step 3 → Done ✓

Blocked Flow:
User → Step 1 → [BLOCKER!] → Fix blocker → Step 2 → Step 3 → Done ✓
                    ↑
            Needs Attention shows this
```

**Visual Relationship:**

```
┌─────────────────────────────────────────┐
│ ⚠️ Needs Attention                      │
│ ┌─────────────────────────────────────┐ │
│ │ API key missing (Blocks step 2)     │ │
│ │ [Fix now →]                         │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘

Checklist:
✓ 1. Choose your agent type
⚠️ 2. Connect tools [BLOCKED - API key needed]
○ 3. Configure behavior
```
