**Needs Attention (Day 0):**

```
Admin logs in → Questionnaire → Admin checklist (0/5)

→ Fixes SSO → Returns to checklist
→ Completes steps 1-5 sequentially
→ Invites team → Reviews security → Done!
```

**Success Metric:** First user logs in with correct access

### 3. ✅ Addressed "Needs Attention" Design Questions

**Question:** Why do we need "Needs Attention" on Day 0? Could this be part of the checklist?

**Answer:** They serve different mental models and should remain separate.

**Rationale:**

| Aspect               | Checklist                    | Needs Attention                      |
| -------------------- | ---------------------------- | ------------------------------------ |
| **Purpose**          | Sequential steps to complete | Urgent blockers to resolve           |
| **Mental Model**     | "Do these in order"          | "Fix these NOW or you'll be blocked" |
| **Priority**         | Proactive, planned           | Reactive, urgent                     |
| **Lifecycle**        | Static, same for everyone    | Dynamic, unique to each user         |
| **Visual Treatment** | Neutral, progressive         | Red banner, high priority            |

**Example Scenario:**

**Without "Needs Attention":**

```
User: *Clicks "Connect your tools" in checklist*
System: "You need API credentials first"
User: "Why didn't you tell me this earlier?!"
```

**With "Needs Attention":**

```
User: *Sees "Set up API connections (2 pending)" in red banner*
User: *Fixes connections first*
User: *Then proceeds to checklist step 2 smoothly*
```

**Design Principle:**

> "Show blockers prominently so users can clear the path before starting the journey."

**Could It Be Integrated?**

We considered integration:

```
Checklist:
⚠️ Prerequisites (2 items need attention)
  - Set up API connections → Fix now
  - Configure guardrails → Fix now

1. Choose your agent type
2. Connect your tools (requires prerequisites)
```

**Problems with integration:**

- Breaks sequential flow
- Mixes urgent with sequential
- Harder to scan for blockers
- Prerequisites might not map 1:1 to steps

**Recommendation:** Keep separate, but consider:

- Linking checklist steps to related "Needs Attention" items
- Auto-dismissing "Needs Attention" items when resolved
- Showing "Needs Attention" count in header

### 4. ✅ Addressed Checklist Side Panel Question

**Question:** Why is the checklist similar to a side panel?

**Answer:** Persistent visibility is critical for progress tracking.

**Rationale:**

**✅ Pros of Side Panel:**

1. **Always Visible** - Users always know where they are (0/5 steps)
2. **Persistent Context** - Doesn't disappear when working in main area
3. **Clear Separation** - Checklist = what to do, Main area = where to do it
4. **Intercom Pattern** - Proven to work in production
5. **Progressive Disclosure** - Can expand/collapse steps without losing place

**❌ Cons of Integration:**

1. Checklist would scroll away as user works
2. Progress would be hidden when deep in a task
3. Harder to see "what's next" at a glance
4. More context switching required

**Design Principle:**

> "The checklist is your map, the main area is your workspace. Keep the map visible while you work."

**Alternatives Considered:**

- Top banner with progress → Too easy to ignore, takes horizontal space
- Integrated cards → Loses persistent visibility
- Bottom progress bar → Can't see next steps

**Future Enhancement (v4.1):**

- Make it collapsible for more workspace
- Add mini progress indicator in header when collapsed
- User control over visibility

### 5. ✅ Renamed "Control Plane Agent" to "watsonx Assistant"

**What was done:**

- Updated v4.0 prototype HTML
- Changed panel title from "Control Plane Agent" to "watsonx Assistant"
- Updated tab labels to be more action-oriented

**Old Names:**

```
Control Plane Agent
├── 🤖 Agent
├── 💬 Help
├── 📰 News
└── 👤 Support
```

**New Names:**

```
watsonx Assistant
├── 🤖 Build
├── 📚 Learn
├── 📰 Updates
└── 💬 Support
```

**Why "watsonx Assistant"?**

| Criteria      | Rating     | Notes                                          |
| ------------- | ---------- | ---------------------------------------------- |
| Brand Aligned | ⭐⭐⭐⭐⭐ | Uses watsonx brand                             |
| Clear Purpose | ⭐⭐⭐⭐⭐ | "Assistant" implies help across multiple areas |
| Multi-Purpose | ⭐⭐⭐⭐⭐ | Doesn't limit to just chat                     |
| Familiar      | ⭐⭐⭐⭐   | Users know what an "assistant" does            |
| Scalable      | ⭐⭐⭐⭐⭐ | Can add more tabs without name change          |

**Why Action-Oriented Tab Labels?**

- **Build** (not "Agent") - Clearer action verb
- **Learn** (not "Help") - More positive framing
- **Updates** (not "News") - More specific
- **Support** (not "Contact") - Clearer purpose

**Benefits:**

- Shorter labels (easier to scan)
- Clearer intent (action-focused)
- Better hierarchy (verbs > nouns)
